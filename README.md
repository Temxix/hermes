# Hermes Agent — Docker Deployment Guide

Инструкция по развертыванию **Hermes Agent** через Docker & Docker Compose.

---

## 📁 Структура файлов

- `Dockerfile`: Образ для сборки Hermes Agent на базе Python 3.11.
- `docker-compose.yml`: Конфигурация запуска сервиса и монтирования томов памяти.
- `.env`: Файл с ключами доступа (OpenRouter API key).

---

## 🚀 Быстрый запуск

### 1. Сборка и первичная настройка

Соберите Docker-образ:
```bash
docker compose build
```

Запустите первичный интерактивный мастер настройки Hermes:
```bash
docker compose run --rm hermes setup
```
В мастере укажите провайдер **OpenRouter** и ваш ключ. Конфигурация сохранится в том памяти `root/.hermes`.

---

### 2. Варианты работы

#### Вариант А: Интерактивный диалог в консоли (CLI)
Если вы хотите общаться с агентом прямо из терминала:
```bash
docker compose run --rm hermes
```

#### Вариант Б: Фоновый запуск (Gateway 24/7)
Если агент подключен к Telegram / Discord или работает в режиме фонового сервера:
```bash
docker compose up -d
```

Проверить логи фонового процесса:
```bash
docker compose logs -f
```

Остановить фоновый процесс:
```bash
docker compose down
```

---

## 💾 Сохранение памяти и навыков

Все данные агента (память, истории сессий, авто-созданные навыки) сохраняются в том `hermes-data`, поэтому они **не удаляются** при пересборке или остановке контейнеров.
